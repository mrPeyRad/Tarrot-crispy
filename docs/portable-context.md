# Перенос проекта на другой ПК с контекстом Codex

## Что важно понимать

Сам проект уже лежит на съёмном диске, но локальный контекст Codex хранится отдельно, обычно в:

```text
C:\Users\<имя>\.codex
```

Там лежат:

- активные сессии;
- архив сессий;
- локальная память;
- локальная индексная и sqlite-состояние Codex.

## Что нужно для максимально полного переноса

1. Тот же OpenAI-аккаунт на новом ПК.
2. Тот же путь к проекту, если это возможно.
   Для этого лучше оставить диск и папку как сейчас:
   `E:\CodexProj\TG_Tarrot_bot`
3. Архив локального контекста Codex.

## Как собрать архив контекста

Закрой Codex на текущем ПК и выполни:

```bash
python scripts/create_portable_context_bundle.py
```

Архив появится в папке:

```text
backups/
```

По умолчанию туда не попадают локальные auth-файлы. Это безопаснее.

Если нужен именно полный локальный перенос на свой доверенный ПК, можно использовать:

```bash
python scripts/create_portable_context_bundle.py --include-auth
```

## Как восстановить на другом ПК

1. Подключи съёмный диск.
2. Постарайся сохранить тот же путь проекта:
   `E:\CodexProj\TG_Tarrot_bot`
3. Установи Codex.
4. Войди в тот же OpenAI-аккаунт.
5. Полностью закрой Codex.
6. Распакуй содержимое папки `codex/` из архива в локальную папку нового ПК:

```text
C:\Users\<имя>\.codex
```

7. Запусти Codex и открой проект.

## Что даст лучший результат

- одинаковый аккаунт;
- одинаковый путь к проекту;
- закрытый Codex во время экспорта и импорта;
- перенос архива контекста вместе с самим проектом.

## Ограничение

Это максимально практичный способ сохранить локальный рабочий контекст Codex между двумя твоими машинами.
Но абсолютную идентичность UI-состояния лучше считать best effort, поэтому важные инженерные решения всё равно полезно фиксировать в коде, коммитах и документации проекта.
